Glossary PRD Architecture

Domain Terms

TermDefinition
ClientAn adult family home (AFH) using the bot. Each client has its own Telegram chat, Google Drive folder, and config entry in config.json.
AFH AdministratorThe primary human user at the AFH facility who sends documents to the bot and confirms classifications via inline buttons. The bot's confirmation flow is designed around their phone-based workflow. Each client has at least one AFH administrator (e.g., Sandra Namwase at Edmonds Villa).
Omega Digital AdministratorThe operator of this bot. Owns the deployment, manages the server, onboards new clients by adding their config and creating their Telegram bot token. Interacts with the bot via systemd, git, and the filesystem rather than the Telegram interface.
DocumentA single file sent to the Telegram channel — photo, PDF, or DOCX. Each processed independently even if sent in a batch.
New EmployeeA name detected in a document that doesn't match any folder in the client's Drive roster. Triggers automatic folder creation and onboarding flow.
Compliance GuideA public reference page explaining WAC 388-76 and 388-112A personnel record requirements for AFHs. Useful for understanding what documents are required and common DSHS citation patterns. → View Compliance Guide

Architecture

DecisionChoice
Bot RuntimeSingle Python script using python-telegram-bot. One systemd service per Telegram bot token (test + production), each sharing the same config.json.
Document ClassificationTwo-tier: (1) Claude Haiku LLM classifies every document — handles name variations, OCR noise, and document context without hardcoded rules; (2) if LLM fails → manual correction. After extraction, bot shows extraction status (✅ Document text found: N chars or ❌ Could not read) with identified employee name and document type before entering the correction flow.
OCR / Text Extractiondoc-extract.py. Uses PyMuPDF + Anthropic Haiku vision for scanned docs. HEIC/HEIF via pillow-heif.
Drive AuthPer-client Google service accounts. Each client has their own SA key with Editor access scoped to their Drive folder only. Eliminates cross-tenant blast radius.
Confirmation UXInline keyboard buttons [👍 Yes] [👎 No]. Correction via two sequential text prompts. Only entered when a pending state exists for that chat.
Employee RosterScanned from Drive folder names on first document request, then cached in-memory with 30-minute TTL. Auto-refreshes on cache miss. Eliminates per-document Drive API calls.
State PersistenceSQLite database (pending.db in data/). All pending classifications survive crashes and restarts. Documents older than 24 hours cleaned on startup.
State MachinePer-chat ChatSession with explicit ChatState enum. asyncio.Lock per chat prevents race conditions on concurrent transitions.
ObservabilityEach document gets a doc_xxx UUID at pipeline entry, threaded through all log lines. Enables per-document tracing.

WAC Categories — Required Documents

Washington Administrative Code chapters 388-76 and 388-112A define the personnel record requirements for AFHs. Each category below lists the specific documents that must be on file per WAC 388-76-10198 and related sections.

01
Identity & Employment
  • Government-issued photo ID (driver's license, passport, or state ID)
  • Employment application (signed and dated)
  • I-9 Employment Eligibility Verification
  • W-4 tax withholding form
  • Staff contact information (address, phone)
  • Work permit (if applicable)
02
Background Check
  • Washington State name/date-of-birth background check (valid 2 years)
  • National fingerprint-based background check (valid indefinitely)
  • Background check disclosure and authorization forms (signed)
  • DSHS background check results letter
  • Portable background check (PBC) number/verification (if applicable)
03
Health Screening (TB)
  • TB test result (Mantoux/PPD skin test or blood test — must be within 3 days of hire)
  • Chest X-ray report (if positive TB test)
  • TB symptom screening questionnaire
  • COVID-19 vaccination record (if applicable)
  • N95 respirator fit test (if required for role)
  • Influenza vaccination or declination form
04
CPR & First Aid
  • CPR certification card (American Heart Association or Red Cross — must be current)
  • First Aid certification card (must be current)
  • AED training certificate (if included in CPR course)
05
Orientation & Training
  • 5-hour safety orientation certificate (completed before providing care)
  • 70-hour basic training certificate (or modified basic training, if applicable)
  • Home Care Aide (HCA) certification
  • Specialty training certificates: dementia, mental health, developmental disabilities (within 120 days if serving residents with special needs)
  • HIV/AIDS bloodborne pathogens training certificate
  • Food handler's permit/ food safety training certificate
  • First aid and CPR training attendance records
06
HCA Certification & Continuing Education
  • HCA license/certification (current, obtained within 200 days of hire)
  • Continuing education records — 12 hours annually by birthday
  • CEU certificates for each training completed
  • Dementia specialty training (7 hours every 2 years if serving dementia residents)
  • Mental health specialty training (if applicable)
  • DSHS specialty training (DDST, if serving residents with developmental disabilities)
07
Nurse Delegation
  • Nurse delegation core training certificate
  • Delegation competency verification (signed by delegating RN)
  • Specific delegated task authorizations (medication administration, insulin, etc.)
  • Annual delegation update/competency check
08
Administrator Training provider only
  • DSHS-approved 54-hour AFH administrator training certificate
  • Administrator competency exam results (quizzes, projects, final exam)
  • Documentation of 1,000 hours direct care experience (per WAC 388-76-10130)
  • High school diploma or GED verification
  • Liability insurance certificate (WAC 388-76-10191)
  • Succession plan (WAC 388-76-10201)
